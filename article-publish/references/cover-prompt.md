# 封面图画图提示规范

封面图和文中的说明性配图（架构图、流程图）目的不同：说明图要把机制讲清楚，封面图要在信息流/列表页里抓住眼球、传达文章主题氛围，不需要承载具体机制细节。调用 **article-image** 生成时，按以下规范写画图提示，而不是套用 `article-image/references/templates/` 里的架构图/流程图模板。

## 核心原则（在 article-image 四条核心原则基础上，封面图额外遵守）

1. **不追求信息完整性，追求第一眼冲击力**：封面图不需要覆盖文章全部机制，选 1 个最能代表主题的意象/场景即可
2. **突出主体，画面简洁**：1 个主视觉焦点 + 少量点缀元素，不要把架构图搬过来当封面——密集的节点和箭头在缩略图尺寸下会糊成一团
3. **不写具体文字标签**：封面图通常不需要节点标注、数据标签这类文字元素（标题另外用 Markdown 文本渲染，不烧进图里）；若确实需要一两个点题的词，控制在极少数、大字号、独立存在
4. **适配缩略图场景**：构图要在缩小到几厘米宽时依然能辨认主体，避免细碎装饰和小字

## 写法

先确定这篇文章的核心主题/隐喻（可以是：核心技术意象的具象化、一个场景/画面、一个抽象概念的视觉化），然后按以下结构写提示：

```
封面主题：[一句话说明这张封面想传达的核心意象]
主视觉：[画面主体是什么，占据画面中心]
氛围/风格：[延用本篇文中配图的风格，或用户指定的封面专属风格]
构图：[横版/竖版，主体位置，留白安排]
```

## 风格选择

默认沿用本篇文中配图已使用的风格（查 `references/preferences.md`「配图偏好」表或文中已有画图提示的 `style:` 字段），保持封面和文中配图视觉统一。用户明确要求封面单独风格时才切换。

## Frontmatter

封面图同样按 article-image `SKILL.md`「与调用方的接口约定」写完整画图提示的 YAML frontmatter（`style`/`ref`/`versions`），但生成的封面图**不写回原文章正文**——frontmatter 和提示内容保存在 `article-publish` 产出的 `meta.md` 里，供下次重新生成封面时复用。

## 调用方式

用 `Skill` 工具调用 article-image，在提示里明确说明"这是文章封面图，不是文中说明图"，并附上按上述结构写好的封面画图提示，让 article-image 走它现有的生成→质检→`AskUserQuestion`确认→上传流程，article-publish 只负责拿到最终确认的 URL 写入 `meta.md` 和发布版正文。
