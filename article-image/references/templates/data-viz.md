## 横向条形图 + 说明卡片

适用：多维度权重或评分分布，每个维度附带机制说明。  
参考图：`../../assets/styles/other/03-weights-bar.png`（6维权重总览）

```
# 画图提示：横向条形图 + 说明卡片布局，展示[N]个维度的权重分布
#
# 左侧：横向条形图，[N]行，每行对应一个维度
#   维度标签（左对齐）：「[维度1]」「[维度2]」…「[维度N]」
#   条形宽度对应权重（从大到小）：[w1] / [w2] / … / [wN]
#   条形颜色：深蓝（最高）渐变到浅灰（最低）
#   每条末尾标注百分比数字
#   左上方标题：「[图表标题]」副标题「[副标题]」
#
# 右侧：[N]张说明卡片，2列排列
#   每卡含：图标 + 维度名（加粗）+ 计算逻辑一句话 + 防作弊机制说明（盾牌图标）
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 左右分栏：公式 + 分级条件卡片

适用：展示公式的分级/分支行为，配折线图直观呈现。

```
# 画图提示：左右分栏布局，左侧展示公式与折线图，右侧展示分级条件卡片。
#
# 左侧：
#   大号公式框：[公式，用中文描述变量]
#   变量说明表：[变量名（中文）] 对应 [含义]
#   折线图（可选）：X轴「[含义]」，Y轴「[含义，范围]」
#   [N]条曲线，标注各自场景名
#
# 右侧：[N]张条件卡片并排
#   每卡：条件标题（「[参数] [≥/</=] [阈值]」）+ 场景名 + 关键参数值
#   卡片颜色与左侧折线一一对应
# 底部说明框：「[一句话总结]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 时间演化折线图

适用：某个指标随时间变化的趋势，含触发阈值线。

```
# 画图提示：折线图，展示[指标名]随时间的演化过程与关键事件节点。
#
# X轴「时间」，Y轴「[指标名]（范围 0~[最大值]）」
# 一条折线，[N]个关键转折点，每点旁标注中文事件：
#   「[事件名]」→ 值=[数值]（超阈值处标「触发[动作]」）
# 水平虚线标注阈值 [M]，旁边标「[阈值含义]」
# 图例：[折线含义] / [虚线含义]
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 金字塔分层图

适用：展示层次递进关系，底层最基础/最宽，顶层最高级/最窄（如优先级金字塔、处理链路分层）。

**⚠️ 金字塔的四条强制规则**（经实践验证，违反任一条都会导致图形扭曲或层次混乱）：

1. **层序方向**：底层 = 第一层（最基础）= 最宽；顶层 = 最高层（最高级）= 最窄的塔尖。**编号从下到上递增**。模型默认容易画反，必须在 prompt 中明确标注「底部最宽为第一层，顶部尖顶为最高层」。
2. **角度必须一致**：三条规则—左侧斜边从底层左角到顶点是一条连续直线；右侧斜边从底层右角到顶点是一条连续直线；所有层的左右两侧均共用同一条斜边，不单独画。违反此规则会导致各层角度不一致，视觉极丑。
3. **必须有塔尖**：顶层要收成三角形的真实尖顶（apex），不是梯形。
4. **不加标题**：金字塔本身不加图表标题，除非用户明确要求。

```
# 画图提示：金字塔分层图，展示[N]层[主题]结构。
#
# 整体形状：等腰三角形，顶部收成尖顶（apex），无图表标题。
# 三角形两侧斜边从底层两角延伸至顶点，是贯通全图的连续直线，各层共用同一斜边。
#
# 从下到上共 [N] 层（底部最宽为第一层，顶部尖顶处为第[N]层）：
#
#   第一层（底部，最宽）：
#     层名标题（加粗）：「[名称]」
#     属性行（带小图标）：
#       📋 处理：[处理描述]
#       ⚙️ 操作：[操作描述]
#       ⏱️ 时间成本：低（[说明]）
#
#   第二层（中部）：
#     层名标题（加粗）：「[名称]」
#     属性行（带小图标）：
#       📋 处理：[处理描述]
#       ⚙️ 操作：[操作描述]
#       ⏱️ 时间成本：中（[说明]）
#
#   第三层（顶部，最窄，含塔尖）：
#     层名标题（加粗）：「[名称]」
#     属性行（带小图标）：
#       📋 处理：[处理描述]
#       ⚙️ 操作：[操作描述]
#       ⏱️ 时间成本：高（[说明]）
#
# 各层配色：三层分别用三种协调色（如蓝/橙/红渐变色系），颜色由模型自行决定。
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```
