# 架构类图型模板

> 属于 article-image 技能的画图参考文档，见 [SKILL.md](../SKILL.md) 总览。

## 横向多层架构图

适用：展示系统分层结构、各层组件和整体数据流。  
参考图：`../../assets/styles/other/01-architecture.png`（MemoryCore 四层架构，重点参考：左侧彩色层名徽章、节点图标+副标题、层间单箭头标语义）

**三区布局规范**（每行一层，左中右三区）：

| 区域 | 内容 | 视觉要求 |
| --- | --- | --- |
| 左区（层名徽章） | 层名文字 | 带颜色背景的圆角矩形，颜色比该行浅色背景深一档 |
| 中区（节点组） | 横向并排节点，每节点：小图标 + 加粗名称 + 灰色副标（1~2行） | 圆角矩形，浅色背景行 |
| 右区（角色描述） | 一句话说这层"是什么角色" | 灰色小字，垂直居中 |

**属性卡片 vs 流程节点**（最容易混淆，必须看清楚再写）：

| 情形 | 节点含义 | 写法 | 箭头 |
| --- | --- | --- | --- |
| 展示该层的**特征/属性** | 并排的属性描述（如：原始消息 / 顺序读取 / 会话级） | 「横向并排三张属性卡片，**卡片之间不用箭头连接**」 | ❌ 无 |
| 展示该层内部的**处理步骤** | 有先后顺序的操作（如：解析→提炼→落库） | 「三个步骤节点，箭头相连」 | ✅ 有 |

错误写法「内容三个节点串联」容易让生图工具在属性列表之间画出箭头，整层变成流程图而非存储架构图。

**层外数据流规范**：

架构图中层与层之间的数据流箭头（如「实时写入」「异步提炼」「混合检索」）必须明确声明在色带**外侧**，不能穿插在层内卡片之间。格式：

```
# 色带外部的数据流（均在色带轮廓外侧，不穿插在节点之间）：
#   顶部：「[来源]」→ 上方色带，标注「[操作]」
#   两层之间：居中垂直箭头，标注「[操作]」
#   右侧：[侧边线说明]
```

**层间箭头规范**：
- 每两层之间只画**一条**居中向下箭头，**不要扇形展开**连到每个节点
- 箭头标注具体操作语义（「调用」「读取 & 写入」「策略查询」），不写"数据流"这类泛称
- 箭头标签下方加白色矩形衬底，防止与背景混叠

```
# 画图提示：横向多层架构总览图，展示[系统名]的[N]层结构与数据流。
#
# 整体布局：[N] 行（每行一层）从上到下排列，每行分三区：
#   左区：层名徽章，带颜色背景的圆角矩形，颜色比该行背景深一档，层名居中
#   中区：节点横向并排，每节点由小图标 + 加粗名称 + 灰色副标组成
#   右区：一句角色定位描述，灰色小字
#
# 行 1【[层名]】（[浅色]背景行，左徽章[深色]）：
#   [N] 个节点，各带图标：
#     「[节点A]」  [副标题]
#     「[节点B]」  [副标题]
#   右区：[一句角色描述]
#
# 行 2【[层名]】（[浅色]背景行，左徽章[深色]）：
#   [N] 个节点，副标题为串联步骤：
#     「[节点C]」  [步骤1] → [步骤2] → [步骤3]
#   右区：[一句角色描述]
#
# 层间箭头（每层之间一条，垂直向下居中，不展开连到各节点，标签有白色衬底）：
#   [层名A] → [层名B]：「[操作语义]」
#   [层名B] → [层名C]：「[操作语义]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 树形层次检索路径图

适用：展示树状结构的遍历/搜索路径，标注每个节点的分数或状态。  
参考图：`../../assets/styles/other/04-tree-search.png`（层次检索路径图）

```
# 画图提示：树形层次检索路径图，展示[查询/搜索]在[树状结构]中的遍历路径与收敛过程。
#
# 外层大框：「[空间名称]（[结构描述]）」
# 树结构（从上到下）：
#   根节点（L0，蓝色高亮）：「[根节点名]（L0，分数 [X]）」
#   L1 子节点（横向排列，4个）：
#     「[目录A]（L1，分数 [低]）」→ 虚线框标「低于阈值，整个子树跳过」
#     「[目录B]（L1，分数 [高]）」→ 继续向下
#     「[目录C]（L1，分数 [高]）」→ 继续向下
#     「[目录D]（L1，分数 [高]）」→ 继续向下
#   L2 叶节点（各高分目录下展开）：
#     各叶节点标注「[文件名]（L2，分数 [X]）」，顶部贴「最终命中」标签
#
# 右侧步骤卡片（3个，从上到下）：
#   步骤1：「[步骤说明]」
#   步骤2：「[步骤说明]」
#   步骤3：「[步骤说明]」
#
# 箭头：实线=实际访问路径，虚线=跳过路径
# 底部图例：各类节点图标含义
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```
