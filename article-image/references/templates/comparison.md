# 对比/并行类图型模板

> 属于 article-image 技能的画图参考文档，见 [SKILL.md](../SKILL.md) 总览。

## 并排方案对比图

适用：两个方案/版本的流程对比，突出差异点。

```
# 画图提示：左右并排对比图，展示[方案A]与[方案B]的差异。
#
# 顶部各贴对比标签（深色背景标题框）：
#   左侧：「[方案A特征，如「无Agent·直连检索」]」
#   右侧：「[方案B特征，如「有Agent·智能路由」]」
# 各自按编号排列步骤：
#   左侧：① → ② → ③（步骤少，流程简单）
#   右侧：① → ② → 三路分支（各分支含关键数字）→ 汇合 → ③ → ④
# 底部横跨两侧对比框：「核心差异：[关键指标对比，如「准确率 +X%（A% vs B%）」]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 并行双路流程图

适用：两条独立路径并行处理同一输入，最终汇合为一个结果（如混合检索、双路校验）。

**核心规范**：
- 两路必须用**不同背景色**区分（如浅蓝 vs 浅橙），顶部各有**深色标题栏**
- 每路的注释/标注文字贴近**该路外侧边缘**，不要放在两路中间
- 底部汇合区用**横跨全宽**的色块（浅绿）表示融合，两路各引一条斜箭头汇入
- 路间没有水平箭头连接，只在底部汇合

```
# 画图提示：并行双路流程图，展示[输入]经两条路径并行处理后汇合的流程。
#
# 顶部：椭圆节点「[输入]」，向下分叉为左右两条路径。
#
# 左路【[路径名A]】（浅蓝背景，圆角矩形，纵向步骤）：
#   顶部深蓝标题栏「[路径名A]」；
#   步骤节点从上到下：「[步骤1]」→「[步骤2]」→「[输出节点A]」（箭头相连）；
#   左侧灰色竖排注释（贴近左边缘）：上行「擅长：[描述]」，下行「盲区：[描述]」。
#
# 右路【[路径名B]】（浅橙背景，圆角矩形，纵向步骤，与左路并排）：
#   顶部深橙标题栏「[路径名B]」；
#   步骤节点从上到下：「[步骤1]」→「[步骤2]」→「[输出节点B]」（箭头相连）；
#   右侧灰色竖排注释（贴近右边缘）：上行「擅长：[描述]」，下行「盲区：[描述]」。
#
# 底部汇合区（浅绿背景，圆角矩形，横跨左右全宽）：
#   左路底端一条斜箭头、右路底端一条斜箭头，同时汇入此区域；
#   区域内：「[融合步骤]」→「[最终输出]」（箭头相连）。
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 左右结果对比图

适用：同一查询/输入在两种方案下的结果对比，重点突出命中 ✓ 与漏掉 ✗ 的差异（如关键词 vs 语义向量检索效果对比）。

与上面的『并排方案对比图』的区别：并排方案对比图比较的是**流程路径**；本节的『左右结果对比图』比较的是**结果列表**，每行结果带 ✗/✓ 图标和右侧小注。

**核心规范**：
- 顶部必须有**共用横幅**作为公共输入，横跨全宽
- 每行结果固定三段式：`图标（✗/✓）+ 内容 + 右侧灰色小注`
- 中央用**垂直灰色虚线**隔开两侧
- 底部各有**颜色标签**总结结论

```
# 画图提示：左右结果对比图，展示[方案A]与[方案B]在同一查询下的结果差异。
#
# 顶部：灰色圆角横幅贯穿全宽，中央文字「查询：[查询内容]」
#
# 左侧面板【[方案A]】（[浅红]背景，圆角矩形）：
#   顶部[红色]标题栏（白色字）：「[方案A名称]」；
#   中间步骤区：小圆角框「[处理步骤]」，向下箭头；
#   结果区（白色背景）：
#     第一行：[红色]叉号 + 「[结果1]」+ 右侧灰色小注「[原因] → [结论]」
#     第二行：[红色]叉号 + 「[结果2]」+ 右侧灰色小注「[原因] → [结论]」
#   底部[红色]圆角标签：「[总结短语]」
#
# 中央：垂直灰色虚线将两个面板隔开
#
# 右侧面板【[方案B]】（[浅绿]背景，圆角矩形）：
#   顶部[绿色]标题栏（白色字）：「[方案B名称]」；
#   中间步骤区：「[步骤1]」→「[步骤2]」（箭头相连），向下箭头；
#   结果区（白色背景）：
#     第一行：[绿色]对号 + 「[结果1]」+ 右侧灰色小注「[原因] → [结论]」
#     第二行：[绿色]对号 + 「[结果2]」+ 右侧灰色小注「[原因] → [结论]」
#   底部[绿色]圆角标签：「[总结短语]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```
