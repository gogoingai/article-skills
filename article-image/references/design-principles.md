# 设计四大原则与配色审美

> 属于 article-image 技能的画图参考文档，见 [SKILL.md](../SKILL.md) 总览。

来源：《写给大家看的设计书》的亲密性/对齐/对比/重复四原则 + 色轮配色法 + 黄金分割构图。以下是转化为 **AI 生图 prompt 写法**的对应规则（不是让模型"理解设计理论"，而是把原则拆成可执行的 prompt 描述）。

### 亲密性——同类归组，异类拉开

同一语义组的节点必须挨在一起、间距一致；不同组之间用更大间距或背景色块分开。

| 已有体现 | 位置 |
| --- | --- |
| 三区布局（层名徽章/节点组/角色描述）左中右各自成组 | templates/architecture.md『横向多层架构图』 |
| 两路并行图左右各自独立背景色包裹 | templates/comparison.md『并行双路流程图』 |
| 手绘图各步骤用不同浅色底色区分 | [styles/handdrawn.md](styles/handdrawn.md) |

**写 prompt 时补一句**：「同组元素间距小而一致，组与组之间间距明显更大」。

### 对齐——看不见的网格线

同一层级的节点必须等宽/等高/顶对齐或中心对齐，不能错落随意摆放。

**已有强制规则**：templates/flow.md『竖向三阶段串行流程图』「步骤节点必须等宽均匀分布」、templates/flow.md『竖向串行流程图（含判断分支）』「所有箭头居中进出」、templates/data-viz.md『金字塔分层图』「所有层共用同一条斜边」。

**通用补充句**：多节点横排或纵排时，prompt 里加「同一行/列的节点顶部对齐，宽度一致」——不要只描述节点内容，不描述对齐关系。

### 对比——让重点跳出来

一张图只能有一个视觉焦点；次要信息必须在颜色饱和度、字号、边框粗细上明显降级，不能和主角平起平坐。

| 场景 | 对比手法 |
| --- | --- |
| 强调"待展开"的卡片（templates/flow.md『两层流程图：主流程 + 展开细节』） | 颜色加深突出，其余卡片保持浅色 |
| 强调"异步阶段"（templates/flow.md『竖向三阶段串行流程图』） | 虚线橙框 + 浅黄底，与同步阶段的实线蓝/绿框区分 |
| 强调图标（手绘风格，[styles/handdrawn.md](styles/handdrawn.md)） | 图标是全图最鲜艳的元素，与浅色底色形成反差 |

**反向提醒**：对比不是"把每个节点都做鲜艳"——多个强调点互相抵消，等于没有重点（参考 pitfalls.md 踩坑 4 的反向陷阱）。

### 重复——同类元素同一套样式

同一层级/同一类型的节点必须共享同一套视觉语言（配色体系、图标风格、圆角半径），不能一个节点方形一个节点圆形、一个深色一个浅色。

**已有强制规则**：templates/flow.md『竖向串行流程图（含判断分支）』「所有节点配色保持同一色调体系」、templates/data-viz.md『金字塔分层图』「各层配色由模型自行决定但三层需协调」。

**通用补充句**：「同类型节点复用同一种卡片样式（同圆角、同图标风格、同色系不同明度）」。

### 配色场景对照表

按画面里出现的颜色种类数选配色策略，不要凭感觉挑颜色：

| 场景 | 策略 | 写法示例 |
| --- | --- | --- |
| 2 种颜色对比（如 templates/comparison.md『并排方案对比图』、templates/comparison.md『左右结果对比图』） | 互补色，一主一强调 | 「左侧红色系（劣势方案），右侧绿色系（优势方案），互为对比色」 |
| 3 种颜色并列（如 templates/data-viz.md『金字塔分层图』三层、templates/comparison.md『并行双路流程图』 双路+汇合） | 等距三色组，愉悦不杂乱 | 「三层分别用蓝/橙/红渐变色系」（已在 templates/data-viz.md『金字塔分层图』采用） |
| 多层/多组（4 种以上，如 templates/architecture.md『横向多层架构图』） | 同色系不同明度，不要彩虹跳色 | 「各层背景色统一为蓝色系，明度从上到下递减」而非「一层蓝一层绿一层紫一层黄」 |

**绝对避免**：任意堆砌 4 种以上互不相关的高饱和色——生图模型会画成"儿童拼图"而非专业配图。

### 黄金分割——已隐含在现有规则里，无需额外声明

- 横向宽屏图的 16:9 构图（见 pitfalls.md 踩坑 5）本身接近黄金比例（1.618:1），不需要再单独强调"黄金分割"。
- 左右分栏类图（templates/data-viz.md『左右分栏：公式 + 分级条件卡片』、templates/comparison.md『并行双路流程图』、templates/comparison.md『左右结果对比图』）默认对半分栏即可；如果一侧是主逻辑一侧是辅助说明（图例/备注），可以按大约 6:4 分配空间而非严格对半，写法：「左侧主图区占约六成宽度，右侧说明区占约四成」。

### 完整感——以终为始（已有体现：见 pitfalls.md 踩坑 4）

画图 prompt 之前先想清楚：**读者只看这张图、没有上下文，能不能看懂？** 检查清单：

- [ ] 图例/标注齐全——特殊符号（虚线、颜色、图标）如果超过一种含义，底部必须有图例说明
- [ ] 标题/子标题能让人一句话知道这张图讲什么（除非按 templates/flow.md『两层流程图：主流程 + 展开细节』/templates/data-viz.md『金字塔分层图』 的规则明确不加标题）
- [ ] 关键数字/阈值/比例已经塞进节点副标题，不是空节点（见 pitfalls.md 踩坑 4）
- [ ] 箭头方向和顺序无歧义（多旁路汇合已按 templates/flow.md『竖向串行流程图（含判断分支）』 四条规则显式声明连线）
