# 奶油描边风格参考库（cream-outline-style）
> 属于 article-image 技能的风格库家族，见 [SKILL.md](../../SKILL.md) 总览；通用画图规范见 [core-principles.md](../core-principles.md)、[pitfalls.md](../pitfalls.md)。


> 参考图存放于 `../../assets/styles/cream-outline/`

第六种手绘/半手绘风格家族，与水彩涂鸦风格（[doodle-watercolor.md](doodle-watercolor.md)）都带手绘感，但视觉语言不同——核心特征是**奶油色/白色卡片 + 黑色粗描边 + 扁平单色图标 + 红蓝弧形箭头**，没有水彩晕染质感，也没有装饰性涂鸦点缀，信息密度更高、更像"知识卡片/思维导图"而非"随手涂鸦"。

| 特征 | 描述 |
| --- | --- |
| 卡片 | 圆角矩形，奶油色或白色底，**黑色粗描边**（线条工整干净，不潦草不抖动） |
| 图标 | 扁平单色/双色渲染（常见蓝色为主，偶尔搭配黄/红/绿），风格多样化（数据库圆柱、齿轮、放大镜、火箭、文件夹等），不要水彩质感、不要手绘涂鸦装饰 |
| 箭头 | 清晰的实线弧形箭头（不是波浪线），颜色常用红/蓝两色区分方向或语义，也可用黑色直线箭头表示单纯顺序关系 |
| 标题 | 黑色加粗无衬线大字，**字号必须适中偏小、单行居中**——这是最容易生成跑偏的地方，不强调会导致标题占满画面 |
| 分支标注 | 判断分支的"是/否"文字必须显式标注在对应箭头旁 |
| 布局 | 灵活多样：可以是线性横向流程，也可以是松散的思维导图辐射布局，也可以是竖向分层 |

**与水彩涂鸦风格（[doodle-watercolor.md](doodle-watercolor.md)）的选择依据**：
- 用户提到「水彩」「涂鸦」「星星火花装饰」「暖色手绘」→ [doodle-watercolor.md](doodle-watercolor.md)
- 用户提到「奶油色卡片」「黑色描边」「思维导图」「知识卡片风」→ 本文件

### 生成规则（--ref + prompt 缺一不可，且 prompt 要简短——本风格的关键踩坑）

**① --ref 必须**：

```
--ref ~/.agents/skills/article-image/assets/styles/cream-outline/cream-outline-01-stage-flow.png       # 横向三阶段流程
--ref ~/.agents/skills/article-image/assets/styles/cream-outline/cream-outline-02-mindmap.png          # 思维导图辐射布局
--ref ~/.agents/skills/article-image/assets/styles/cream-outline/cream-outline-03-layered-arch.png     # 竖向三层架构
--ref ~/.agents/skills/article-image/assets/styles/cream-outline/cream-outline-04-branch-decision.png  # 竖向判断分支流程
```

**② prompt 只写"参考图风格 + 内容"，不要逐条复述视觉细节**（见 `pitfalls.md` 踩坑 9）：

```
参考图风格，[横向宽屏构图 / 竖版构图]。标题字号适中偏小，不要占据画面主导地位，单行居中即可。[内容描述]。所有文字用中文，拼写清晰。
```

**⚠️ 标题字号是本风格唯一需要反复强调的点**：不写"字号适中偏小/单行居中"，模型默认会把标题放大到占据画面主导位置（参考图原图标题本身占比也不小，容易被模型放大复刻）。除此之外不需要再补充图标/箭头/配色的细节描述——写多了反而会把 `--ref` 的风格覆盖掉。

### Prompt 模板片段

**横向多阶段流程** — 参考 `cream-outline-01`：
```
参考图风格，横向宽屏构图。标题字号适中偏小，不要占据画面主导地位，单行居中即可。[N] 阶段流程：[阶段1] → [阶段2] → … → [阶段N]，箭头连接。所有文字用中文，拼写清晰。
```

**思维导图辐射布局** — 参考 `cream-outline-02`：
```
参考图风格，思维导图布局。中央标题「[标题]」（字号适中偏小，不要占满中央区域，留出足够留白）。周围节点：[节点1]、[节点2]、…、[节点N]，节点间用弧形箭头连接。所有文字用中文，拼写清晰。
```

**竖向分层架构** — 参考 `cream-outline-03`：
```
参考图风格，竖版构图。[N] 层架构从上到下：[层1] → [层2] → … → [层N]，层间箭头连接。每层可包含多个子节点。所有文字用中文，拼写清晰，标题字号适中不要太大。
```

**竖向判断分支流程** — 参考 `cream-outline-04`：
```
参考图风格，竖版构图。判断分支流程：[起始节点] → [条件]？菱形判断，是→[是分支后续]→[终点]，否→[否分支结果]。两个分支不要互相连接，分支间保持足够宽的间距。所有文字用中文，拼写清晰，标题字号适中不要太大。
```
